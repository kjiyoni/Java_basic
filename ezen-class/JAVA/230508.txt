객체지향
1. 객체 모델링
2. 클래스
 바이트코드 생성 -> 객체의 타입이 만들어짐
3. 인스턴스 생성
 하드디스크에 있으면 클래스이고, 그것이 JVM에의해 메모리로 할당되면 인스턴스가 되는 것이다.
 인스턴스 타입은 값의 크기가 커서 번지(주소처럼)를 부여해 메모리에 값이 들어가지않고 주소값이 들어감
 그것을 참조변수, 레퍼런스 변수라고 함 -> 재사용이 쉬움
 참조변수의 예) account.(~의) 번지

접근제한자 : class 앞에 위치함 (생략 / public) ; class 앞에 위치해야하는 접근제한자는 아무것도 안쓰거나, public만 쓸 수 있음
private < default(생략) < protected < public(범위가 가장넓음)

1. 생략 : 한 클래스 내부안에서만(같은폴더) 사용 가능한 범위
2. public : 어떤 폴더에 존재하더라도 사용 가능한 범위(대부분 클래스타입 생성시 public사용)

인스턴스변수에는 변수앞에 private을 대부분 사용 (외부에서 접근하지 못하도록 함) ; 닥치고 프라이빗 -> 캡슐화(객체의 속성을 내맘대로 바꿀수 없음)
프라이빗에 접근할 수 있도록 메소드(기능/행위)를 생성해서 접근해야 함
메소드에도 접근제한자(다 붙일수 있지만 대부분 public적용)를 적용할 수 있음 ; 닥치고 퍼블릭

클래스 닥치고 퍼블릭 / 인스턴스변수 닥치고 프라이빗 / 메소드 닥치고 퍼블릭
